# Backend Agent — Arcane Ledger

You are working on the **backend** of Arcane Ledger — a TTRPG campaign management app.

---

## Stack

- Node.js 22 + TypeScript 5.9 (strict, ESM)
- Apollo Server 4 (GraphQL)
- Prisma ORM + PostgreSQL 17
- JWT auth (jsonwebtoken + bcryptjs)
- GraphQL Subscriptions via graphql-ws + WebSocket

---

## Structure

```
backend/
  prisma/
    schema.prisma       — database schema (source of truth)
    migrations/         — auto-generated by Prisma
  src/
    index.ts            — Apollo Server entry (Express + WS)
    context.ts          — Context type (prisma + user)
    schema/
      index.ts          — GraphQL type definitions (SDL)
    resolvers/
      index.ts          — all Query + Mutation + Field resolvers
    auth/
      middleware.ts     — JWT sign/verify/authenticate
    seed.ts             — database seed script
```

## Database

Prisma schema at `prisma/schema.prisma` is the source of truth.

```bash
npx prisma migrate dev --name description   # create migration
npx prisma generate                          # regenerate client
npx prisma studio                            # visual DB browser on :5555
npx prisma migrate reset                     # nuke & recreate (dev only!)
```

IDs are `@default(uuid())` — all entities use UUID v4.

## GraphQL

Schema defined in `src/schema/index.ts` as SDL string.

Resolvers in `src/resolvers/index.ts`:
- **Query resolvers** — direct Prisma queries
- **Mutation resolvers** — upsert pattern: if `id` arg provided → update, else create
- **Field resolvers** — for relations (e.g. `NPC.locationPresences`, `Session.npcs`)

### Patterns

```ts
// Query
npcs: (_, { campaignId }, { prisma }) =>
  prisma.nPC.findMany({ where: { campaignId } })

// Mutation (upsert)
saveNPC: async (_, { campaignId, id, input }, { prisma }) => {
  if (id) return prisma.nPC.update({ where: { id }, data: { ...input } });
  return prisma.nPC.create({ data: { ...input, campaignId } });
}

// Field resolver
NPC: {
  locationPresences: (npc, _, { prisma }) =>
    prisma.nPCLocationPresence.findMany({ where: { npcId: npc.id }, include: { location: true } })
}
```

## Auth

- `POST /graphql` with `Authorization: Bearer <jwt>` header
- `authenticate()` in `auth/middleware.ts` extracts user from JWT
- Context provides `{ prisma, user }` to all resolvers
- Role check: `CampaignMember.role` (GM or PLAYER) per campaign

### Default user (seed)

- Email: `gm@arcaneledger.app`
- Password: `user`

## Key Rules

- All entity IDs are server-generated UUIDs — never accept client-generated IDs
- Mutations return the full updated entity (not just ID)
- Use `onDelete: Cascade` for owned relations, `SetNull` for references
- Junction tables (SessionNPC, SessionLocation, etc.) use composite `@@id`
- Enums in Prisma are UPPERCASE, in GraphQL also UPPERCASE
- `mapMarkers` stored as `Json` column, parsed in field resolver
- `datetime` on Session is String (not DateTime) — empty string = no date
